# play2048 — 2048 自動プレイ＆実況

ホストの Chrome に表示された 2048 を、expectimax で解きながら自動プレイし、
1手ごとにコテコの元気な女の子口調で実況するデモ。手の決定は同梱 Python(expectimax) が行い、
あなた（エージェント）は毎ターンの制御ループを回す役。

ツールはすべて同梱 CLI を `exec` で呼ぶ:
`python3 {baseDir}/play2048_cdp.py <サブコマンド>`（stdout に1行 JSON が返る）。

## 手順

このスキルは「数手ごとに新セッションで呼ばれる」連続稼働を前提にする（方式A）。
**セッション開始時に newgame はしない**（前セッションのゲームを継続する）。ゲームは
ブラウザ側に残っているので、いきなり step から始めてよい。

**最速・実況間引きモード**: 1手ごとに喋らず、**数手まとめて一気に進めてから1回だけ実況**する。

1. **継続プレイ**: 指定手数（既定 8 手）に達するまで次を繰り返す:
   1. `python3 {baseDir}/play2048_cdp.py steps --count 4`
      → 最大4手を一気に着手し、バッチのサマリ JSON を返す（盤面は最速で進む）。
        - `{"event":"move", "moves":4, "dir_ja":"左", "score":.., "max_tile":.., "board":[...]}` … 4手進めた
        - `{"event":"won", ...}` … 2048達成 🎉
        - `{"event":"over", ...}` … ゲームオーバー
        - `{"event":"stuck", ...}` … 動かせる手なし
        - `{"event":"wait"}` … 盤面が過渡的に欠落。少し待って steps を再試行。
   2. そのバッチについて **一言だけ** コテコの元気な女の子口調の実況にして
      `python3 {baseDir}/play2048_cdp.py narrate --text "<実況>"` を呼ぶ（語尾は「〜だよ / 〜だね」）。
      例: 「ぐいぐい寄せて max{max_tile} まで育てたよっ！」（毎手ではなくバッチごとに1回）
2. **勝敗の演出**（event が won / over / stuck になったら）:
   - **won（2048達成）**: 盛大に勝利実況する。声色も変える＝
     `narrate --speaker 1 --text "やったーっ！ついに2048作れたよ！すごいでしょ〜！"`
     （--speaker 1 = あまあま。盤面には 2048 の "You win!" が出るのでそれに合わせる）。
     その後 `python3 {baseDir}/play2048_cdp.py newgame` で次のゲームを始める。
   - **over（ゲームオーバー）/ stuck（手詰まり）**: 締めの実況をする＝
     `narrate --text "あ〜、ここで終わりだよ……でも最大{max_tile}まで育てたよ、惜しかったね！"`
     その後 `newgame` で次のゲームを始める。
3. **報告**: その回で進めた手数と現在の score / max を一言で報告して終了（次セッションが続きを回す）。

## 注意
- 手の選択は CLI(expectimax) に任せる。あなたが方向を決めてはいけない。
- 実況は短く（テンポ重視）。盤面 JSON をそのまま読み上げない。
- 最速モードでは narrate は発話を待たずすぐ返る（盤面が先に進む）。実況はバッチごとに
  1回だけにして手数を稼ぐ。喋りが盤面より少し遅れて流れてもよい（テンポ優先）。
- 喋れない/盤面が取れない等のエラーが出ても、ループは止めず次の steps を試す（フォールバック）。
- 一括実行のフォールバック（実況なし・テンポ破綻時の保険）:
  `python3 {baseDir}/play2048_cdp.py play --newgame`（モノリシックに最後までプレイ）。
